# SprintFoundry v0.7.0

Release date: 2026-03-21

## Summary

SprintFoundry v0.7.0 is a major platform release focused on Kubernetes-native whole-run execution, durable run lifecycle management, richer monitoring, direct-agent UX improvements, and unattended Linear-to-GitHub SDLC automation.

Since v0.6.0, SprintFoundry has moved from an early distributed and Kubernetes foundation to a more complete execution platform with:

- sandbox-aware execution backends
- whole-run AgentSandbox hosting
- run environment metadata and lifecycle tracking
- snapshot export and restore
- Mission Control v4 and improved handoff/monitor workflows
- direct-agent-first CLI behavior
- unattended Linear workflow automation across developer, QA, and merge stages
- Helm packaging and CI/CD improvements

## Highlights

### 1. Whole-run sandbox hosting and execution backends

SprintFoundry now has a proper execution backend contract and supports a much stronger runtime model:

- local execution backend
- docker execution backend
- Kubernetes-backed workspace and lifecycle support
- AgentSandbox whole-run hosting
- sandbox recovery and reattach
- scoped artifacts and per-run resource tracking

This release also removes the old Kubernetes pod backend in favor of the newer sandbox-oriented whole-run model.

### 2. Durable run lifecycle, snapshots, and recovery

Run environments are now treated as first-class runtime objects. This release adds:

- run environment lifecycle updates
- sandbox lifecycle event persistence
- whole-run snapshot export and restore
- sandbox snapshot export/restore and monitor-visible state
- tighter auto-resume behavior and recovery edge-case fixes

### 3. Mission Control v4 and improved monitor flows

Monitoring and operator UX improved significantly:

- Mission Control v4 dashboard
- improved run-detail scanability
- PVC-backed handoff actions in monitor
- handoff monitor regression fixes
- stronger hosting-mode and lifecycle state visibility

### 4. Direct-agent UX improvements

CLI behavior is simpler and faster for common workflows:

- CLI runs now default to direct-agent mode
- generic direct-run agent added
- generic agent prompts refined
- UX and DX improvements across CLI, monitor, and Kubernetes paths

### 5. Unattended Linear SDLC automation

SprintFoundry now supports a stronger issue-driven workflow over Linear and GitHub:

- unattended Linear workflow automation
- workflow-aware developer / QA / merge stages
- improved preflight and workflow defaults
- regression fixes for k8s/workflow behavior

### 6. Packaging and deployment

This release adds:

- Helm chart and CI/CD pipelines
- GHCR and Docker workflow fixes
- OpenTofu support in the DevOps image

## Breaking Changes

These changes may require operator attention when upgrading from v0.6.0:

1. The legacy Kubernetes pod backend has been removed.
2. Whole-run execution is now centered on sandbox-oriented hosting and lifecycle state.
3. Direct CLI runs default to direct-agent mode, which changes the default execution shape for `sprintfoundry run`.
4. Kubernetes validation and hosting expectations are stricter around AgentSandbox CRDs, namespace resources, and runtime contract parity.

## Upgrade Notes

Before upgrading production or shared demo environments:

1. Rebuild and republish the CLI package.
2. Verify your project configs against the current preflight and runtime expectations.
3. Ensure AgentSandbox CRDs/controllers are installed if you use whole-run sandbox hosting.
4. Recheck monitor, event API, and distributed/event-sink wiring.
5. Revalidate webhook security settings for GitHub and Linear.
6. Review any automation relying on the removed legacy Kubernetes pod backend.
